## 2024-05-24 - [Avoid Multiple String Allocations]
**Learning:** Utilizing multiple calls to `strlen()`, `strcpy()`, and `strcat()` sequentially to build strings leads to unnecessary multiple passes over the strings, increasing CPU cycles. This pattern was identified in several URL redirection handlers.
**Action:** Replace `strcpy` and `strcat` loops or sequences with a single `snprintf()` call into a correctly sized buffer. When dynamic memory is involved, precalculate the exact needed size once, then execute `malloc` and `snprintf`, freeing up variables right after.
