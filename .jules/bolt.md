## 2024-05-19 - Eliminate O(N) strlen() overhead in static handlers
**Learning:** ESP-IDF's `httpd_resp_send` with `HTTPD_RESP_USE_STRLEN` macro calls `strlen()` on the entire payload. For embedded static files (like CSS, JS, HTML), this is an unnecessary O(N) operation since the file size is already known at compile time via linker symbols (e.g., `_binary_styles_..._css_end`). `favicon_get_handler` correctly uses pointer arithmetic to precalculate the size (`end - start`), but `styles_download_get_handler`, `jquery_get_handler`, and `reset_get_handler` do not.
**Action:** Replace `HTTPD_RESP_USE_STRLEN` with pre-calculated sizes using linker symbol pointer arithmetic (`(size_t)(end - start) - 1`) for embedded static assets to save CPU cycles and improve response time.
