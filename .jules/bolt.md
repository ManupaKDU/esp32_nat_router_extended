## 2024-05-11 - Eliminate O(N) strlen() overhead for static embedded files
**Learning:** In ESP-IDF, serving large static embedded files (like jQuery or CSS) via `httpd_resp_send` with the `HTTPD_RESP_USE_STRLEN` macro forces the HTTP server to invoke `strlen()` internally. This scans the entire file payload byte-by-byte to determine its length on *every single request*, which is extremely inefficient (O(N)) for static content on an embedded microcontroller.
**Action:** When serving embedded files (where CMake `EMBED_TXTFILES` automatically appends a null terminator), replace `HTTPD_RESP_USE_STRLEN` with pointer arithmetic using the linker-generated symbols: `(size_t)(file_end - file_start) - 1`. This computes the exact payload length in O(1) time at compile/link time, completely bypassing the runtime `strlen()` overhead. Additionally, use `0` instead of `HTTPD_RESP_USE_STRLEN` for explicitly empty `""` payloads.
