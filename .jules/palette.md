## 2024-05-15 - Visual Feedback for Long-Running Operations
**Learning:** The application uses `<meta http-equiv="refresh">` and polling to simulate wait states for long-running operations like reboots (`apply.html`) and network scans (`scan.html`). These pages previously lacked clear visual indicators of ongoing background activity, which can lead to user confusion or premature navigation. Furthermore, adding new UI components (like Bootstrap's `spinner-border`) requires manually appending their CSS definitions to the purged, minified CSS file (`styles-[hash].css`) to ensure they render correctly.
**Action:** When improving pages that rely on polling or delayed redirects, always include an accessible loading indicator (e.g., `<div class="spinner-border text-primary" role="status"><span class="visually-hidden">Loading...</span></div>`). Ensure any new Bootstrap classes used are present in the project's minified CSS file, appending them if necessary.
